6.5 MB — the map itself
Page design + drawing code + the default scenario's numbers written directly inside it
scenarios/s0021.json … (43 files)
~5 MB each
The same distillation, one file per scenario

The scripts keep only what gets drawn — 766 rivers & canals, 62 reservoirs, 223 delivery areas — simplify the shapes, and round the numbers. About 2% of the original size.

git pushfrom your Mac (or via a bundle / your signed-in browser)

3 · The public shelf

GitHub Pages — free file hosting, no computation

GitHub simply hands out copies of the finished files to whoever asks. There is no server "thinking" — if GitHub is up, the map is up. Nothing changes here until you push again.

one visitthe visitor downloads the page once (6.5 MB)

4 · The visitor's browser

Where all the animation happens — on their computer, not yours
opens the map
Default scenario s0020 is already inside the page — works immediately
picks another scenario
Fetches just that one file (~1–2 MB compressed); A/B keeps two in memory
background map
The only outside pieces: basemap tiles (CARTO) and the Leaflet drawing library
The key idea: every number a visitor ever sees was computed in advance by the build scripts. GitHub is a shelf of finished files, and the visitor's own browser is the projector. That is why hosting is free, nothing needs maintenance, and the map cannot "go down" on its own — and also why adding a new scenario means running a script and pushing one new file.
