Ordinal vs. continuous tiers

Methods note for the sector tier teams: how to convert integer tier assignments into continuous tier values.

Provenance: recovered 2026-06-07 from a chat-generated PDF, saved as-is beside this file (ordinal_vs_continuous_tiers.pdf). This markdown file is the living copy; it applies the 2026-06-07 review fixes (current tier labels, two added subsections, harmonized notation). Companions: tier_continuous_conversion.xlsx (the per-sector worksheet applying this framework — its Decision column is still unfilled) and coeqwal_framing_invariants.md (the axis rules as used by visualization work). Status: the COEQWAL API serves integer tier levels only; the data-platform team has per-location continuous tiers in progress, and the briefs and visualizations are built to consume them as a data-only change.

The task

Tiers are currently defined by integers (1–4), but are calculated from continuous data. Rather than binning tiers into the four integer values, we want to preserve the continuous nature of the underlying values used in tier calculations and generate continuous tier values ranging between 1 and 4.99. The tier definitions and thresholds remain the same, but the thresholds — the values where the teams judged that qualitative character changes — now become anchor points on that continuous axis.

The task for each tier team is to modify their tier calculations to allow variation in values within each tier. This will make it possible for the axes on which tier values are plotted to include tick marks, like a ruler, instead of being separated into categorical bins.

Axis convention: [1, 4.99]

All tier values should be calculated on a continuous axis of [1, 4.99], with equal band widths:

TierBandPublic label
11.0 up to (not including) 2.0Optimal
22.0 up to (not including) 3.0Acceptable
33.0 up to (not including) 4.0At-risk
44.0 through 4.99Critical

Each band includes its lower edge and excludes its upper edge, so every value belongs to exactly one tier: 1.999 is Tier 1, exactly 2.0 is Tier 2. (In interval notation: Tier 1 = [1.0, 2.0), Tier 2 = [2.0, 3.0), Tier 3 = [3.0, 4.0), Tier 4 = [4.0, 4.99].)

We are intentionally staying away from 5 to avoid introducing a fifth tier. Under this convention the best possible outcome is 1.0 and the worst possible outcome is 4.99. The integer at the bottom of each band is the best possible outcome within that tier, and floor(continuous value) recovers the discrete tier: a location with a continuous value of 2.73 is Tier 2; a location at 3.01 is Tier 3.

(The PDF's figures annotate the bands as "1.0–1.99" etc. — same convention, but that style leaves values like 1.995 unassigned; the include-the-bottom, exclude-the-top rule above is the precise version.)

Linear vs. nonlinear mapping

There are two approaches for mapping continuous values within each band, depending on the nature of the underlying metric.

Linear mapping

In the simplest case, an underlying continuous variable used in tier calculations scales evenly across the full range of outcomes. A linear mapping distributes values uniformly within each tier band: the midpoint of the raw metric range for a given tier maps to the midpoint of that band, and so on. This is appropriate when the relationship between the raw metric and outcomes is roughly proportional and when the team has no reason to weight one part of a tier's range more heavily than another.

Figure 1 (see the PDF): linear mapping — the raw metric maps to the continuous tier value with a constant slope across the full range.

Nonlinear mapping

For some sectors, the relationship between the underlying variables and the tier values may not be uniform across tier states. For example, Tier 1 for agricultural revenue (AG_REV) is defined by a change in revenue above 0%, whereas Tier 2 is defined as a loss of less than 5% and Tier 3 as losses between 5% and 20%. In this case the relationship between the continuous tier value and changing revenue is nonlinear (piecewise-linear or curved). The slope of the mapping function varies: steep where small changes in the raw metric correspond to large differences in outcomes, gradual where a change in the metric is less consequential.

Figure 2 (see the PDF): nonlinear mapping — the slope varies across the range, reflecting that the significance of a unit change in the raw metric is not constant.

Whether to use a linear or nonlinear mapping depends on the tier definitions and the nature of the data used in tier calculations.

The threshold constraint

For some sectors, the tier assignment is threshold-driven: if a location does not meet the gate for a tier, it falls to the next one regardless of how close it was. This means a location whose raw metric would naively rescale to 2.9 might actually be Tier 3 because it failed the Tier 2 gate. In those cases, the continuous value cannot be 2.9 — it must be ≥ 3.0 to land in the correct band.

The mapping function must therefore be built in two steps for threshold-driven sectors:

  1. The tier logic determines which band the location belongs to.
  2. The raw metric determines where within that band the location sits.

For sectors where a naive rescale and the gate logic agree, the mapping is simple. For sectors where they diverge (particularly those with compound conditions or shifting measurement targets across tiers) the mapping must be piecewise, built from the tier assignment outward.

The key constraint: the continuous value must reproduce the correct discrete tier when floor() is applied. If it does not, the two representations tell contradictory stories about the same location.

Positioning within an open-ended Tier 4

(Added 2026-06-07; not in the original PDF.)

Several sectors have no lower bound on how bad outcomes can get — the conversion worksheet flags an open-ended Tier 4 for most sectors (e.g. agricultural production loss has no floor). The [4.0, 4.99] band still needs a defined interior, so each affected team must choose how to position results within it: for example, mapping a stated worst case to 4.99 and scaling within the band, or saturating — beyond some reference severity, everything reads 4.99. Whichever rule a team picks should be recorded in its methods, because downstream users will read distances within Tier 4 as meaningful.

Percentile-based sectors need a fixed reference

(Added 2026-06-07; not in the original PDF.)

The Delta ecosystem composite is percentile-ranked (top 25% / top 50% / top 75%). Percentiles are relative to a comparison set: if the ranking is recomputed whenever the scenario library grows, every continuous tier value in the sector shifts even though no model result changed. Percentile-based sectors should rank against a fixed, named reference distribution (a stated scenario set and date) so continuous values stay stable and comparable across releases.

Worked example: environmental flows (ENV_FLOWS / eFlows)

The eFlows sector illustrates several of the challenges described above: compound conditions, shifting measurement targets across tiers, and the need for a piecewise mapping. Walking through this sector in detail shows how the general principles apply in practice.

Tier structure

eFlows assigns tiers based on how well modeled flows satisfy functional flow targets across seasons. Each tier requires meeting both a flow magnitude condition and a seasonality condition. The flow targets also differ with each tier:

  • Tier 1 (Functional Ecosystem): seasonal flows exceed all functional flow targets in at least 95 of 100 years, and seasonality is preserved in every year.
  • Tier 2 (Dry Year Functional Flow Minimums): flows meet dry-year functional flow targets in at least 90 of 100 years, and seasonality is preserved in every year.
  • Tier 3 (Minimal Baseflows): flows exceed dry-year baseflow targets in at least 90 of 100 years, and seasonality is preserved in at least 95 of 100 years.
  • Tier 4 (No Function): conditions for Tiers 1–3 are not met.

(These tier names are the eFlows team's working labels for their tier definitions, distinct from the public Optimal / Acceptable / At-risk / Critical labels.)

Two things make this sector non-trivial for continuous mapping. First, the flow target itself changes between tiers: Tier 1 uses the full suite of functional flow targets, Tier 2 uses dry-year functional flow targets, and Tier 3 uses only baseflow targets. Second, each tier requires satisfying both flow magnitude and seasonality conditions simultaneously.

The raw metric

The eFlows team computes, for each location, the number of years (out of 100) that satisfy the flow target for each respective tier. This gives a single continuous metric (years satisfying target) but the target itself shifts depending on which tier is being evaluated.

Figure 3 (see the PDF): years satisfying the flow target for each respective tier, mapped to continuous tier values, for an example set of locations. Note the figure is an Excel chart screenshot with a stray "Plot Area" tooltip; regenerate it when the method is finalized.

Several features of this mapping are worth noting:

  • Within each tier band, the mapping is approximately linear: more years satisfying the target means a better (lower) continuous value within that band.
  • At the tier boundaries, there are visible discontinuities. A location that just barely satisfies all Tier 1 conditions jumps into the Tier 1 band, even if its year count for the Tier 2 target would have placed it well within the Tier 2 band.
  • The slope and range of the mapping differ between tiers because the year-count ranges are different for each tier's flow target. Most locations cluster in the Tier 4 band with relatively low year counts, while locations that reach Tier 1 have very high year counts against a more demanding target.
  • The mapping is piecewise by necessity: because the flow target changes at each tier boundary, a single linear function across the full range would not reproduce the correct tier assignments.

Handling the compound condition

For eFlows, the seasonality condition acts as a secondary gate. A location that meets the flow magnitude criterion for Tier 2 but fails the seasonality criterion falls to Tier 3 (or lower). In the continuous mapping, this means the location's continuous value is determined by its position within the Tier 3 band, not the Tier 2 band — regardless of how well it scored on flow magnitude alone.

The eFlows team addresses this by evaluating the compound condition first to assign the correct tier band, then using the flow magnitude metric to position the location within that band. This two-step approach is the general pattern for any sector with compound conditions.

Qualitative tier labels

The qualitative tier labels (Optimal, Acceptable, At-risk, Critical) remain available as regions of the axis, communicated through background shading or annotation on visualizations. A user looking at a location at 2.7 reads it as "in the Acceptable band, closer to At-risk than to Optimal." The interpretive framework is unchanged; we now get to keep the within-tier variation that was previously lost by binning.

(A note on terms: this document says "location" where older drafts said "node" — these are the model's Locations of Interest, LOIs. The original PDF also used the retired tier-2 label "sub-optimal"; the public labels are Optimal / Acceptable / At-risk / Critical per shared/coeqwal_conventions.md.)
